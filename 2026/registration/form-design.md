# 参加申込フォーム設計

## 概要

Google フォームで作成予定。申込データはスプレッドシートに自動集計され、バンドマッチングに使用する。

---

## フォーム項目

### 基本情報

| 項目 | 形式 | 必須 | 備考 |
|---|---|---|---|
| お名前（ニックネーム可） | テキスト | ✓ | |
| X（Twitter）アカウント | テキスト | | @なしで入力 |
| GitHubアカウント | テキスト | | |

### パート

| 項目 | 形式 | 必須 | 備考 |
|---|---|---|---|
| 第一希望パート | 選択 | ✓ | ボーカル / ギター / ベース / ドラム / キーボード / その他 |
| 第二希望パート | 選択 | | 「なし」含む |

### 音楽性

| 項目 | 形式 | 必須 | 備考 |
|---|---|---|---|
| 希望ジャンル | 複数選択 | ✓ | 邦ロック / 洋ロック / ポップス / ファンク / メタル / アコースティック / なんでもOK |
| 好きなアーティスト・バンド | テキスト（自由記述） | ✓ | 3〜5組程度。マッチングの参考にします |
| やってみたい曲・避けたいこと | テキスト（自由記述） | | |

## マッチングロジック（案）

申込データをもとに以下の優先順でバンドを組む：

1. **ドラムを起点に** バンド数を決定
2. ドラムの **希望ジャンル・好きなアーティスト** をもとにバンドの方向性を仮設定
3. 各バンドに **希望ジャンルが近い** 他パートをアサイン
4. **活動エリア・練習可能時間帯** の重複が多いメンバーを優先
5. パート不足の場合は第二希望パートで補完

マッチングはハナ（AI）が複数案を生成 → ジョシセが確認・調整 → 参加者間で最終調整
