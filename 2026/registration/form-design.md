# 参加申込フォーム設計

## 目標

- 参加者から不満が出ない（マッチング結果に納得感がある）
- 演奏経験によらず参加しやすい（初心者でも疎外感がない）
- バンドを組んだときに楽しめる（練習が成立して本番が盛り上がる）

| 好きなアーティスト・バンド | テキスト（自由記述） | ✓ | 3〜5組程度。マッチングの参考にします |
| やりたい音楽の雰囲気 | テキスト（自由記述） | | 例: 明るくノリノリな感じ、エモくてドラマチックな感じ |

## マッチングロジック

バンドマッチングはLLM（ハナ）が申込データを読んで実行する。
プログラムによる決定的なアルゴリズムは使わない。

### 優先順位

**最優先: バンドが成立すること**
- 各バンドに必ずドラムが1人いる（ドラムの人数＝バンド数）
- ボーカル・ギター・ベースが揃っている
- パート不足は第二希望で補完。それでも不足する場合は報告

**第2優先: 練習が成立すること**
- 活動エリアが重なっている
- 練習できる時間帯が週1コマ以上重なっている
- 練習頻度の希望が大きくズレていない（月1希望と週1希望を同じバンドにしない）

**第3優先: 音楽的に楽しめること**
- 好きなアーティストの雰囲気が近い
- やりたい音楽の方向性が一致している

**第4優先: 初参加者への配慮**
- 各バンドにTRF経験者を最低1人配置
- 演奏歴が極端に偏らないようにする（全員1年未満のバンドを作らない）
- 経験者が初心者をサポートするという上下関係にはしない

### 出力形式

ハナは各バンドについて以下を出力する：
- メンバー一覧（名前・パート）
- 編成の理由（2〜3文）
- 懸念点があれば併記

### ハナへの指示プロンプト（ひな形）

```
以下の申込データをもとにバンドを編成してください。

【制約】
- ドラムの人数分のバンドを作る
- 各バンドにボーカル・ギター・ベース・ドラムを揃える
- パート不足は第二希望で補完（それでも不足する場合は報告）
- 各バンドにTRF経験者を最低1人入れる
- 演奏歴が極端に偏らないようにする

【優先順位】
1. 練習が成立する（エリア・時間帯・頻度が重なる）
2. 音楽の方向性が近い（好きなアーティストをもとに判断）
3. 経験レベルのバランスが取れている

【出力形式】
各バンドについて以下を出力してください：
- メンバー一覧（名前・パート）
- 編成の理由（2〜3文）
- 懸念点があれば併記

【申込データ】
（ここにJSON形式で全員分のデータを貼る）
```
